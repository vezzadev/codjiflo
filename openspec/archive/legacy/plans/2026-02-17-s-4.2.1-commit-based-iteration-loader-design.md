# S-4.2.1 Commit-Based Iteration Loader - Design

**Date**: 2026-02-17
**Story**: S-4.2.1 from Milestone 4.2 (Stateless Iteration Management)
**Goal**: Enable iteration tracking for PRs without the CodjiFlo GitHub Action by mapping each PR commit to an iteration and detecting force-push history via the Timeline API.

---

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Type model | Unified `Iteration` type | Extend existing `Iteration` with optional `status` and `collapsedGroupId`. Downstream consumers work unchanged. |
| Store integration | Extend `loadIterations` | Same action, two paths (artifact vs timeline). Single entry point. |
| Scope boundary | Full data model + basic UI | Build complete `CollapsedIterationGroup` structures. Render collapsed tabs in IterationSelector (grayed out, non-interactive). |
| Architecture | Loader class | `TimelineLoader` follows `ArtifactLoader` pattern. Encapsulates API calls + iteration building. |

---

## Type Extensions

### Extended `Iteration` interface (`types.ts`)

```typescript
export type IterationLifecycle = 'live' | 'collapsed';

export interface Iteration {
  id: number;
  revision: number;
  headSha: string;           // commitSha maps here for stateless
  baseSha: string;
  beforeSha: string | null;
  author: string;
  createdAt: Date;
  status: IterationLifecycle; // NEW: 'live' (default) or 'collapsed'
  collapsedGroupId?: string;  // NEW: links to CollapsedIterationGroup
}
```

Stateful iterations get `status: 'live'` as default. No breaking changes.

### New types

```typescript
export interface CollapsedIterationGroup {
  forcePushEventId: string;
  discardedRevisions: number[];
  commits: DiscardedCommit[];
  reason: 'force_push';
  visibility: 'collapsed' | 'expanded';
  unknownCount?: boolean;
}

export interface DiscardedCommit {
  sha: string;
  message: string;
  author: string;
  date: string;
  status: 'available' | 'unavailable';
}

export interface TimelineLoaderResult {
  iterations: Iteration[];
  collapsedGroups: CollapsedIterationGroup[];
}
```

## TimelineLoader Class

**File**: `src/features/iterations/timeline-loader.ts`

### Responsibilities

1. Fetch PR commits via `GET /repos/{owner}/{repo}/pulls/{pr_number}/commits`
2. Fetch PR timeline via `GET /repos/{owner}/{repo}/issues/{pr_number}/timeline`
3. Extract `head_ref_force_pushed` events from timeline
4. For each force-push, call Compare API to discover discarded commits
5. Handle 404/410 on Compare (GC'd commits -> `unknownCount: true`)
6. Build iteration list: discarded as `collapsed`, current as `live`
7. Sort chronologically, assign sequential revision numbers

### API Types (private)

```typescript
interface PRCommit {
  sha: string;
  commit: { message: string; author: { name: string; date: string } };
  author: { login: string } | null;
}

interface TimelineEvent {
  id: number;
  event: string;
  created_at: string;
  before_commit?: { sha: string };
  after_commit?: { sha: string };
}
```

### Class Shape

```typescript
class TimelineLoader {
  constructor(owner, repo, prNumber, baseSha) {}

  async load(): Promise<TimelineLoaderResult> {
    const [commits, timeline] = await Promise.all([
      this.fetchCommits(),
      this.fetchTimeline(),
    ]);
    return this.buildIterations(commits, timeline);
  }

  private fetchCommits(): Promise<PRCommit[]>
  private fetchTimeline(): Promise<TimelineEvent[]>
  private discoverDiscardedCommits(afterSha, beforeSha): Promise<DiscoveryResult>
  private buildIterations(commits, timeline): TimelineLoaderResult
}
```

### Key Behaviors

- Commits and timeline fetched in parallel
- Force-push discovery is sequential per event (each needs Compare API call)
- `id` field: use negative IDs (`-revision`) for stateless iterations
- `beforeSha`: set to force-push `before_commit.sha` for collapsed iterations
- `headSha`: maps from `commit.sha` for both live and collapsed iterations

## Store Integration

### New state field

```typescript
collapsedGroups: CollapsedIterationGroup[];
```

### Modified `loadIterations` flow

```
loadIterations()
  -> try artifact loading
  -> if artifact found -> stateful mode (unchanged, set status:'live' on iterations)
  -> if no artifact -> call TimelineLoader.load()
    -> if iterations found -> stateless mode WITH iterations + collapsedGroups
    -> if error -> stateless mode with error message
```

### Default range selection

Same as stateful: base -> latest live iteration.

## IterationSelector Changes

### Remove stateless early return

Currently returns `null` when `mode === 'stateless'`. Change to render when `iterations.length > 0`.

### Collapsed group tabs

- Group consecutive collapsed iterations into a single tab per `collapsedGroupId`
- Apply `.collapsed` CSS class (grayed out)
- Show Eraser icon from lucide-react
- Tooltip: "N iterations discarded"
- Non-interactive in this story (click behavior deferred to S-4.2.2)
- Skipped during drag-to-select range

## Testing Strategy

### Unit: TimelineLoader

- Simple PR (3 commits, no force-pushes)
- PR with 1 force-push (discoverable discarded commits)
- PR with force-push where before SHA is GC'd (404 on Compare)
- Empty PR (no commits)
- Multiple force-pushes with mixed discoverable/GC'd

### Unit: Store integration

- Verify `loadIterations` falls through to TimelineLoader when no artifact
- Verify iterations populated with correct `status` field
- Verify `collapsedGroups` stored correctly

### Unit: IterationSelector

- Collapsed tabs render with `.collapsed` class
- Collapsed tabs show Eraser icon
- Collapsed tabs skipped during range selection
- Mix of live and collapsed iterations renders correctly

### E2E: Mock mode

- Load PR with mocked commits/timeline endpoints
- Verify iteration selector shows correct number of tabs
- Verify collapsed group tab appears with correct tooltip
