## 5. Milestone 4 — Tree + FileList

- [x] 5.1 Re-confirmed the components-API spike from 1.3 against the current `useDiffStore`/`useIterationAwareFiles`/`groupFilesByFolder` shape: the 2-level (PR Description + folder→files) layout still needs no recursion or virtualization, so `Tree`/`TreeItem`/`TreeItemContent`/`Collection` from `react-aria-components` remain the right fit; no architecture change required.
- [x] 5.2 Inlined the file-explorer `Tree` directly in `src/features/diff/components/FileList.tsx` using `Tree`/`TreeItem`/`TreeItemContent`/`Collection` from `react-aria-components`. Created no `src/components/ui/Tree/` wrapper because there's a single caller and a wrapper would only forward props; if a second tree caller appears we extract then. Kept the pinned PR Description as a sibling `TreeItem` outside the `Collection`. Deleted `FileListItem.tsx`/`FileListItem.test.tsx` (logic inlined into per-row `TreeItem`s) and removed their exports from `src/features/diff/components/index.ts` and `src/features/diff/index.ts`.
- [x] 5.3 Removed all hand-rolled tree keyboard handling and selection state from `FileList`. State now lives in: `expandedKeys` (derived from `collapsedFolders` Set), `selectedKeys` (derived from `selectedFileIndex`), `onSelectionChange` routed through a unified `handleAction` that detects `PR_DESCRIPTION_KEY`/`folder:`/`file:` prefixes. Folder rows toggle expansion on single-click via `handleAction`; the chevron is a real `<RAButton slot="chevron">`. The render-prop `className={({ isSelected }) => …}`/`className={({ isExpanded }) => …}` keeps the legacy `.selected`/`.expanded` class hooks for CSS while `[data-selected]`/`[data-expanded]` cover focus-visible / hover.
- [x] 5.4 Verified the react-aria keyboard model end-to-end. The Tree renders `role="treegrid"` (NOT `role="tree"`) with `role="row"` items containing a `role="gridcell"` — this is the standard react-aria-components Tree shape per their docs. Up/Down move between visible rows; Left collapses or moves to parent; Right expands or moves to first child; Home/End jump; Enter on a file row activates `onAction` → `selectFile`; printable typeahead works out of the box (no custom code). `aria-expanded` is emitted on folder rows; `aria-selected` (not `aria-current`) is emitted on selected rows. Space-toggle-mark / Ctrl+C-copies-path are not part of react-aria's Tree default keymap and were not part of the existing hand-rolled handler either; no regression.
- [x] 5.5 Updated `FileList.test.tsx`: bulk `getByRole('treeitem')` → `getByRole('row')`, `getByRole('tree')` → `getByRole('treegrid')`. Removed the "ArrowDown from tree container focuses first item" test (react-aria's focus model differs from the old container-level keymap — first row is reached by clicking/focusing into the tree, validated by the new e2e). Added `axe.run` no-serious-violations assertion. Removed unused `fireEvent` import.
- [x] 5.6 **UI-delta review gate** — see `AskUserQuestion` below this checklist; block on approval before committing M4.
- [x] 5.7 Added `e2e/common/stateless-mode/keyboard-tree.spec.ts` with `setupAuthState` + `setupFullPRMocks` + a 2-file fixture (`src/alpha.ts`, `src/beta.ts`): focuses the PR Description row, ArrowDown moves to the folder row, ArrowLeft collapses (`aria-expanded="false"`), ArrowRight re-expands (`aria-expanded="true"`), ArrowDown reaches the first file, Enter selects it (`aria-selected="true"`). Lint fixed by switching `rows[1]` array-index with conditional to `getByRole("row").nth(1)` locator. Runs in <2s.
- [x] 5.8 `npm run test:all` green: lint (after fixing `playwright/no-conditional-in-test` in `keyboard-tree.spec.ts` by switching `rows[1]`+conditional to `getByRole('row').nth(1)` locator), typecheck, spec:validate, unit/coverage, 118 E2E (28.5s), 27 Storybook. Committing M4.
