# Questions - 2026-03-25

## Context

While reviewing `spec/functional` files to identify user journeys that need E2E tests, I found several spec-defined features that don't appear to be implemented in the codebase. This document lists clarification questions about these gaps.

---

## 1. Unauthenticated Comments (unauthenticated-access.md)

The spec defines a "Comment Interaction Flow" for unauthenticated users:

> - All comments display in read-only mode
> - Reply button shows "Log in to reply"
> - Add comment button hidden OR shows login prompt on click

**Current state:** The `CommentThread`, `CommentEditor`, and `CommentPortalManager` components don't check authentication state. They always render the full interactive UI (reply input, edit/delete buttons) regardless of auth status.

**Questions:**
1. Is the read-only comment experience for unauthenticated users a planned feature that hasn't been implemented yet?
2. Should I file a separate issue for implementing this feature?

---

## 2. Collapsed Iterations Click Behavior (iterations-stateless.md)

The spec defines this behavior for collapsed iteration tabs:

> **Click behavior:**
> - Replaces diff area with history view showing:
>   - Each discarded iteration with number and commit message
>   - "Include discarded iterations" button
> 
> **When included (visibility = `expanded`):**
> - Collapsed tab expands to full-size individual tabs

**Current state:** The `CollapsedGroupTab` component renders a non-interactive tab with a tooltip. It has no onClick handler and doesn't implement the "history view" or expansion functionality. The type system has `visibility: 'collapsed' | 'expanded'` but the expanded state doesn't appear to have UI support.

**Questions:**
1. Is the collapsed iteration expansion feature planned but not yet implemented?
2. Should I file a separate issue for this?

---

## 3. Bubble Comment Mode (comments.md)

The spec describes two comment display modes:

> - **Bubble Mode** (primary): Floating comment boxes at the right edge, connected to code via "lasso" lines
> - **Interlinear Mode**: Comments displayed inline between code lines

**Current state:** Only interlinear mode appears to be implemented. Comments are rendered inline within the CodeMirror editor using block widgets.

**Question:**
1. Is bubble mode a future feature, or is interlinear mode sufficient for the current milestone?

---

## Summary

All specified user journeys that ARE implemented already have E2E test coverage. The gaps I found are at the feature implementation level, not the test coverage level.
