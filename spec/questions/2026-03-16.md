# Questions from Janitor Task - 2026-03-16

## Context

While reviewing the folder structure as part of janitor task #2026-03-16, I analyzed the `src/features/iterations` folder and found an inconsistency with the documented conventions in AGENTS.md.

## Observation

The `src/features/iterations` folder uses a hybrid structure:

1. **Clean Architecture pattern** for SpanTracker:
   - `application/` - SpanTrackerService (use cases, port interfaces)
   - `domain/` - SpanTracker, LineSpan, SpanMapping (pure domain logic)
   - `infrastructure/` - SQLiteSpanTrackerReader (adapters)

2. **Standard feature pattern** (per AGENTS.md):
   - `components/` - IterationSelector
   - `stores/` - useIterationStore
   - `types.ts` - Feature types

3. **Root-level loader files** that don't fit either pattern:
   - `artifact-loader.ts` - Downloads artifacts from GitHub Actions API, manages IndexedDB cache
   - `iteration-client.ts` - SQLite query interface for iterations/artifacts
   - `timeline-loader.ts` - Fetches iteration data from GitHub Timeline API

Other features (`auth`, `pr`, `diff`, `comments`, `keyboard`, `theme`) consistently follow the standard pattern documented in AGENTS.md section 1.1.

## Questions

1. **Was the Clean Architecture pattern for SpanTracker an intentional architectural decision?**
   - If yes, should this be documented in AGENTS.md as a valid exception for complex domain logic?

2. **Should the root-level loaders be organized consistently?**
   - Option A: Move to `infrastructure/` alongside SQLiteSpanTrackerReader
   - Option B: Keep at root level but document this as a valid pattern
   - Option C: Create a separate `loaders/` subdirectory

3. **Is the hybrid structure (Clean Architecture + standard feature pattern) intended to coexist?**
   - This might confuse a junior engineer who reads only AGENTS.md

## Recommendation

If the Clean Architecture pattern is intentional, I suggest one of:
1. Update AGENTS.md to document when Clean Architecture subdirectories are appropriate
2. Move the Clean Architecture subdirectories under `span-tracker/` to isolate them from the standard structure
3. Consolidate all infrastructure concerns under `infrastructure/`

No changes were made during this janitor task - awaiting clarification on the intended architecture.
