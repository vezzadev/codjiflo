# Milestone 4.2: Stateless Iteration Management - Design Document

**Date:** 2026-01-31
**Status:** Approved
**Spec:** [spec/stories/milestone-4.2-stateless-iteration-management.md](../../spec/stories/milestone-4.2-stateless-iteration-management.md)

## Overview

Upgrade Stateless Mode to achieve near-parity with Stateful Mode without requiring the GitHub Action. Enable iteration tracking, cross-iteration diffs, and comment persistence using only GitHub's native APIs.

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| API loading strategy | Eager with caching | Most PRs have <100 events; simpler code |
| Web Worker RPC | Comlink | Type-safe, ~3KB, spec requirement |
| IndexedDB wrapper | idb | Minimal (~1KB), sufficient for 3 stores |
| Collapsed UI expansion | Inline | Stays in context, matches spec |
| Worker architecture | Single worker | Simpler scheduling, sufficient for typical PRs |
| SpanTracker trigger | Schedule immediately at Low priority | Scheduler handles ordering naturally |
| Git state field | `lineage: 'current' \| 'discarded'` | Separates git state from UI visibility |
| Tracing | OTel-aligned structured JSON | Future observability integration |
| Live demos | pedropaulovc/codjiflo-e2e-test-repo | Real GitHub data, per-scenario folders |

## Architecture

### High-Level Component Diagram

```
┌─────────────────────────────────────────────────────────────────┐
│                     Iteration Loading Layer                      │
├─────────────────────────────────────────────────────────────────┤
│  TimelineLoader  ←──→  CommitLoader  ←──→  IterationBuilder     │
│  (force-push events)   (PR commits)       (builds iteration list)│
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     Diff Computation Layer                       │
├─────────────────────────────────────────────────────────────────┤
│  DiffScheduler  ←──→  DiffComputeWorker  ←──→  ResultCache      │
│  (priority queue)     (Comlink + Web Worker)  (in-memory)        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     Persistence Layer                            │
├─────────────────────────────────────────────────────────────────┤
│  StatelessStorage (idb)                                          │
│  - lastSeen: {prKey → iterationRevision}                        │
│  - iterations: {prKey/revision → commitSha, lineage}            │
│  - unavailable: {prKey/revision → reason}                       │
└─────────────────────────────────────────────────────────────────┘
```

### Data Flow

1. PR loads → TimelineLoader fetches events → IterationBuilder creates `StatelessIteration[]`
2. UI renders iteration selector
3. User selects file → Scheduler queues diff at Highest priority
4. Worker computes → UI displays
5. Background: SpanTrackers computed for comment-containing files at Low priority

## Key Interfaces

### StatelessIteration

```typescript
interface StatelessIteration {
  revision: number;                      // Sequential across all commits
  commitSha: string;
  baseSha: string;                       // Parent commit or PR base
  author: string;
  message: string;                       // First line of commit message
  createdAt: Date;
  lineage: 'current' | 'discarded';      // Git state (not UI state)
  collapsedGroupId?: string;             // Groups discarded commits
}
```

### CollapsedIterationGroup

```typescript
interface CollapsedIterationGroup {
  id: string;
  beforeSha: string;                     // SHA before force-push
  afterSha: string;                      // SHA after force-push
  iterations: StatelessIteration[];
  visibility: 'collapsed' | 'expanded';  // UI state (in-memory only)
  unavailableReason?: string;            // If Compare API returns 404/410
}
```

### DiffTask / DiffResult

```typescript
interface DiffTask {
  taskId: string;
  filePath: string;
  leftSha: string;
  rightSha: string;
  compareMode: '2dot' | '3dot';
  signal?: AbortSignal;
}

interface DiffResult {
  taskId: string;
  status: 'completed' | 'cancelled' | 'error' | 'unavailable';
  diffLines?: DiffLine[];
  alignedLines?: AlignedLine[];
  unavailableReason?: '404' | '410';
  error?: string;
}
```

### DiffScheduler

```typescript
enum DiffPriority {
  Highest = 0,   // Currently selected file
  High = 1,      // Files in user-selected iteration range
  Medium = 2,    // Current iteration to latest
  Low = 3,       // SpanTrackers, background precomputation
}

interface DiffScheduler {
  schedule(task: DiffTask, options: ScheduleOptions): string;
  prioritize(taskId: string): void;
  clear(): void;
  getResult(taskId: string): DiffResult | undefined;
  onComplete(callback: (result: DiffResult) => void): () => void;
}
```

### IndexedDB Schema

```typescript
// Database: 'codjiflo-stateless', version 1

interface LastSeenRecord {
  prKey: string;                 // "{owner}/{repo}/{number}"
  iterationRevision: number;
  headSha: string;
  timestamp: number;
}

interface IterationRecord {
  key: string;                   // "{prKey}/{revision}"
  prKey: string;
  revision: number;
  commitSha: string;
  baseSha: string;
  lineage: 'current' | 'discarded';
  collapsedGroupId?: string;
  discoveredAt: number;
}

interface UnavailableRecord {
  key: string;                   // "{prKey}/{revision}"
  prKey: string;
  revision: number;
  reason: string;                // "404" | "410" | "unknown"
  detectedAt: number;
}
```

## Implementation Order

```
Phase 1 - Foundation:    S-4.2.1 (Loader), S-4.2.9 (Query Param)
Phase 2 - Persistence:   S-4.2.6 (IndexedDB)
Phase 3 - Async Infra:   S-4.2.3 (Worker), S-4.2.4 (Scheduler)
Phase 4 - Integration:   S-4.2.5 (Pipeline), S-4.2.8 (Unavailable)
Phase 5 - UI Polish:     S-4.2.2 (Collapsed UI)
Phase 6 - Background:    S-4.2.7 (SpanTracker)
```

### Parallel Opportunities

- S-4.2.6 can run parallel with S-4.2.3
- S-4.2.9 is independent
- S-4.2.2 can start once S-4.2.1 types are defined
